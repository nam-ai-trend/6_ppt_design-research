# 기술 발표자료 스타일 PPTX 스킬
> 첨부된 `~/ppt_design_test/executive-minimal-templete.pptx`를 시각적 기준으로 참조하여 동일한 완성도로 생성하세요.

---

이 스킬은 **엔터프라이즈 AI 기술 문서 및 B2B 발표 자료**에 최적화된 프레젠테이션을 생성합니다.

**높은 대비(블랙/화이트/밝은 회색)의 모던한 카드형 레이아웃**을 바탕으로, 블루 액센트와 깔끔한 타이포그래피를 사용하여 **전문성과 기술적 신뢰감(Tech-Authority)**을 강조하는 디자인이 특징입니다.

**트리거 키워드:** 'AI 기술 자료', '기술 발표', '엔터프라이즈 보고서', '아키텍처 설명', '플랫폼 소개', '기술 제안서', '테크 브리핑', '연구소 리포트', '시스템 구성도', 한국어로 전문적인 IT/기술 발표 자료 요청 시 이 스킬을 최우선으로 적용하라.

### 공통 뱃지 규칙 (모든 슬라이드)

모든 슬라이드의 상단 뱃지("CONCEPT", "INTELLIGENCE STRATEGY" 등)는 동일한 스타일로 적용.

- **도형**: `ROUNDED_RECTANGLE`, `rectRadius: 0.05`, h=0.28"
- **텍스트**: 12pt, `#FFFFFF`, 중앙 정렬
- **배경**: `#18181B` fill
- **위치**: x=0.55", y=0.6" (모든 슬라이드 공통)
- **w 자동계산 공식** (글자 수에 따라 동적):
  ```javascript
  const badgeW = badgeText.length * 0.1 + 0.5;
  // 예시: "CONCEPT" (7자) → 0.7 + 0.5 = 1.2"
  // 예시: "INTELLIGENCE STRATEGY" (23자) → 2.3 + 0.5 = 2.8"
  ```

### 이미지 및 아이콘 규정

#### 1. 시각적 이미지 생성 (Tech-Editorial & Academic 하이브리드)
프레젠테이션에 사용되는 모든 이미지는 다음의 **[이미지 생성 3단계 프로토콜]**을 엄격히 준수하여 생성합니다.

**[이미지 생성 3단계 프로토콜 - 필수 준수]**
1.  **Priority 1 (에이전트 내장 도구)**: 에이전트(나)는 반드시 자신의 내장 도구(`generate_image`)를 사용하여 모든 에셋 생성을 우선 시도합니다. 이는 사용자의 무료 쿼터를 최우선으로 활용하기 위함입니다.
2.  **Priority 2 (쿼터 초과 시 폴백)**: 내장 도구의 쿼터가 모두 소진되었을 경우에만, 사용자에게 상황을 보고하고 외부 API(Gemini 2.5 Flash 등)를 사용하는 폴백 모드로 전환합니다.
3.  **Priority 3 (스크립트 소비)**: 최종 출력용 스크립트(`generate_ppt.js`)는 가급적 생성된 이미지를 **로드**하는 역할에 집중하며, 이미지 생성을 직접 수행할 때는 반드시 위 우선순위가 반영된 로직을 포함해야 합니다.

- **스타일**: 3D 렌더링(Tech-Editorial) 또는 학술 도해(Academic) 스타일을 슬라이드 성격에 맞춰 하이브리드로 적용.
- **비율**: **1:1 정사각형** (본문용), 16:9 또는 Bleed(타이틀용)

**[학술 도해 강제화 규칙 (Strict Academic Figure Rule)]**
- 논문, 연구 리포트, 기술 아키텍처 등 학술적/기술적 신뢰도가 필요한 슬라이드에서는 단순히 멋진 3D 이미지가 아니라 **'학술 도해(Academic Diagram)'** 스타일의 이미지를 강제로 생성해야 함.
- `generate_image` 호출 시 프롬프트에 `schematic diagram, scientific chart, clean minimalist architecture diagram, black and white with blue accent, scientific paper figure` 등의 키워드를 명시하여 논문 그림 같은 완성도를 확보할 것.
- 만약 데이터나 구조를 설명하는 내용이라면, 추상적인 이미지 대신 순서도나 시스템 구성도 형태의 도해를 생성하도록 프롬프트를 유도할 것.



#### 2. 아이콘 활용
- **SVG 기반**: stroke가 아닌 **fill 속성**이 포함된 SVG를 사용하거나, `sharp`를 통해 PNG로 래스터라이즈하여 삽입.
- **이모지 사용 금지**: 전문성을 위해 이모지 대신 고품질 아이콘 또는 3D 에셋 이미지를 사용함.

|--------------|------------|------|------|
| **타이틀** | Bleed (w=4.75", h=5.17") | x=5.25", y=0" | 배경형 이미지 |

| **카드형** | 아이콘 또는 미니 이미지 | 카드 상단 중앙/좌측 | SVG/PNG |

### 공통 레이아웃 세이프존

- **좌측 여백**: 모든 텍스트/요소는 x=**0.55"** 이상에서 시작
- **하단 세이프존**: 본문 콘텐츠는 구분선(y=5.17") 위로 최소 **0.2"** 여유 → 콘텐츠 최대 y = **4.97"**
- **콘텐츠 가용 영역**: x=0.55" ~ 9.45", y=1.0" ~ 4.97"


### 아이콘 규칙

- **이모지 사용 절대 금지**
- 모든 아이콘은 SVG를 base64로 인코딩하여 `slide.addImage({ data: "image/svg+xml;base64,..." })` 방식으로 삽입할 것.
- **함수 사용**: `function svgB64(svgStr) { return "image/svg+xml;base64," + Buffer.from(svgStr).toString("base64"); }`
- **크기**: 일반 카드/스텝 아이콘은 `w:0.3", h:0.3"` 유지.
- **색상 동적 적용 (매우 중요)**:
  - 밝은 배경(기본 흰색 또는 연한 쿨그레이 카드) 위: `fill="#000000"` 또는 액센트 컬러 `fill="#2563EB"`
  - 어두운 배경(다크 강조 카드 또는 섹션 간지) 위: 반드시 흰색 `fill="#FFFFFF"` 적용

### 타이틀 슬라이드 우측 이미지 규칙 (아이콘 아님)

- 타이틀 슬라이드의 우측 절반 영역에는 대형 SVG 아이콘 대신, **보고 주제와 관련된 고해상도 실사 이미지(사진)**를 삽입.
- **배치 방식**: 슬라이드 우측 영역에 꽉 차게(Bleed) 배치하거나, 좌측 텍스트 영역과 명확히 구분되는 레이아웃 사용.
- **예시 규격**: `x: 5.0, y: 0, w: 5.0, h: 5.625` (슬라이드 크기에 맞춰 우측 절반을 덮도록 설정)

### 글머리 기호 규칙

- **이모지 사용 금지**
- **내장 bullet 방식** 사용 — 별도 도형(사각형) 방식 사용 금지
- 텍스트박스 안에서 `bullet: { indent: 12 }` 옵션으로 적용
- 줄 간격: `paraSpaceAfter: 8~10` 으로 조정
- 여러 항목은 rich text 배열로 구성, 마지막 항목 제외 `breakLine: true`

```javascript
// ✅ 올바른 내장 bullet 사용법
slide.addText(
  items.map((text, i, arr) => ({
    text,
    options: { bullet: { indent: 12 }, breakLine: i < arr.length - 1 }
  })),
  { x: 0.5, y: 1.0, w: 3.5, h: 2.0,
    fontFace: "Black Han Sans", fontSize: 15, color: "9B9B9B",
    valign: "top", margin: 0, paraSpaceAfter: 8 }
);
```

### 본문 내용 좌측 여백 규칙

- 카드/박스 내부 텍스트 시작: 박스 좌측 경계로부터 최소 **0.2" + 0.2" (0.5cm) = 0.4"** 이상
- 글머리 기호: 박스 좌측 경계로부터 `+0.37"` (indent 0.2" 반영)
- 텍스트: 글머리로부터 `+0.12"` 간격

### 타이틀/마감 슬라이드 텍스트 규칙

- 쉼표, 마침표 앞 공백 없이 붙여 쓸 것: `디지털 전환,` (O) / `디지털 전환 ,` (X)
- 줄바꿈이 필요한 경우 **별도 텍스트박스**로 분리하여 렌더링 오차 방지

- **상단 타이틀 영역 (y: 0.5" ~ 1.2")**: 
  - 좌측에 주제 분류를 나타내는 둥근 다크 뱃지(Padding 적용) 삽입

   ```javascript
   const LIST_TOP = 2.08;      // 결론 아래 0.4"
   const LIST_BOTTOM = 4.97;   // 세이프존
   const ITEM_HEIGHT = 0.6;    // 제목 + 설명 1세트 높이
   ```

   > 모든 카드: y=1.28", h=3.69", 가용 영역 너비=8.9" (0.55" ~ 9.45")

   | 단수 | 카드 w | 간격 | x1 | x2 | x3 | x4 | x5 |
   |------|--------|------|-----|-----|-----|-----|-----|
   | **2단** | 4.30" | 0.30" | 0.55" | 5.15" | - | - | - |
   | **3단** | 2.77" | 0.30" | 0.55" | 3.62" | 6.69" | - | - |
   | **4단** | 2.04" | 0.25" | 0.55" | 2.84" | 5.13" | 7.42" | - |

   **동적 계산 공식** (n단 카드):
   ```javascript
   const AREA_LEFT = 0.55;
   const AREA_WIDTH = 8.9;
   const GAP = n <= 3 ? 0.3 : n === 4 ? 0.25 : 0.2;
   const cardW = (AREA_WIDTH - GAP * (n - 1)) / n;
   const cardX = (i) => AREA_LEFT + i * (cardW + GAP);
   // 카드 내부 텍스트 w: cardW - 0.6" (좌우 0.3" 패딩)
   ```

   **카드 색상 규칙**:
   - 기본: `#F4F4F5` (라이트)
   - 마지막 카드 또는 핵심 강조 카드: `#18181B` (다크)
   - 2단: 좌=라이트, 우=다크 / 3단 이상: 마지막 1개만 다크 (또는 전부 라이트)

   **카드 내부 요소** (⚠️ 텍스트가 카드 가장자리에 붙지 않도록 반드시 내부 여백 확보)

   | 요소 | 카드 내 위치 | 크기 | 색상 | 비고 |
   |------|------------|------|------|------|
   | 아이콘 | 좌상단 +0.3", +0.3" | w=0.3", h=0.3" | SVG | 어두운 카드: `#FFFFFF` |

   ```javascript
**[빈 공간 방지 및 이미지 배치 규칙 (Anti-Blank Space Rule)]**
- 카드 내부 텍스트가 적어 하단에 큰 빈 공간(약 0.5" 이상)이 생길 경우, 텍스트만 두고 비워두지 말 것.
- **해결책 A**: 카드 본문 텍스트 아래나 우측 여백에 내용과 관련된 미니 이미지(1:1 비율)나 도해를 추가하여 공간을 채울 것.
- **해결책 B**: 텍스트 양이 극히 적다면 카드형 대신 '기본 1:1 이미지형(4번 타입)' 레이아웃으로 변경하여 좌측엔 텍스트, 우측엔 큰 이미지를 배치함으로써 공간 활용도를 극대화할 것.

   ```

   **카드형 변형 A: 하이라이트 인용 블록 + 카드** (결론 강조 후 카드 배치)

   | 카드 시작 y | - | 인용 끝 + 0.3" ≈ **2.28"** | - | - | - | |
   | 2단 카드 | 0.55" / 5.15" | 2.28" | w=4.3", h=2.69" | - | `#F9F9F9` fill | RECTANGLE |

   **카드형 변형 B: 스펙 그리드 (2열 × N행)**

   뱃지 + 제목(공통 헤더) 아래에 2열로 소형 스펙 카드를 다행 배치. 기술 스펙, 사양 정보 등에 적합.

   | 요소 | x | y | w | h | 색상 | 비고 |
   |------|---|---|---|---|------|------|
   | 카드 (좌열) | 0.55" | *동적* | 4.3" | 0.85" | `#F9F9F9` fill | RECTANGLE, 좌측 accent border |

   | 값 | 카드 +0.3" | 카드 +0.45" | - | **16pt** | `#000000` | Black Han Sans |

   **스펙 그리드 동적 배치** (행 수 rows에 따라):
   ```javascript

   const CARD_H = 0.85;
   const ROW_GAP = 0.2;
   ```

| 슬라이드 번호 | 권장 타입 | 설명 |

| 4 ~ N-2 | 콘텐츠 (혼합) | **'기본 1:1 이미지형'을 메인으로 사용**하되, 내용에 따라 '카드형(고대비)', '비교/스텝형'을 적절히 혼합하여 배치 |

| N-1 | 데이터/요약 | 테이블(표) 객체나 핵심 KPI 수치를 활용한 전체 요약 및 결론 |
